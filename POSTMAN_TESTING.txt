Пошаговая

Пример userUid: 834532c2-2a8e-4d43-b245-908c438ded7b

1. Запустиьприложение
2. Убедится,  что обе базы данных (transaction_service_db1 и transaction_service_db2)
3. Откройте Postman

ШАГ 1. СОЗДАНИЕ ТИПА КОШЕЛЬКА
--------------------------------------
POST http://localhost:8080/api/v1/wallet-types
Тело запроса (JSON):
{
  "name": "Основной",
  "currencyCode": "USD",
  "status": "ACTIVE"
}
Ответ: uid типа кошелька (скопируйте его)

ШАГ 2. СОЗДАНИЕ КОШЕЛЬКА
--------------------------------------
POST http://localhost:8080/api/v1/wallets
Тело запроса (JSON):
{
  "userUid": "<ВАШ userUid>",
  "name": "Мой кошелек",
  "walletTypeUid": "<UID типа кошелька из шага 1>"
}
Ответ: uid кошелька (скопируйте его)

ШАГ 3. ПОЛУЧЕНИЕ ВСЕХ КОШЕЛЬКОВ ПОЛЬЗОВАТЕЛЯ
--------------------------------------
GET http://localhost:8080/api/v1/wallets?userUid=<ВАШ userUid>
Ответ: список кошельков

ШАГ 4. ПОЛУЧЕНИЕ КОШЕЛЬКА ПО ВАЛЮТЕ
--------------------------------------
GET http://localhost:8080/api/v1/wallets?userUid=<ВАШ userUid>&currencyCode=USD
Ответ: список кошельков с валютой USD

ШАГ 5. СОЗДАНИЕ ПЛАТЕЖНОГО ЗАПРОСА
--------------------------------------
POST http://localhost:8080/api/v1/payment-requests
Тело запроса (JSON):
{
  "userUid": "<ВАШ userUid>",
  "walletUid": "<UID кошелька>",
  "amount": 100.00,
  "comment": "Пополнение",
  "paymentMethodId": 1
}
Ответ: uid платежного запроса (скопируйте его)

ШАГ 6. СОЗДАНИЕ ТРАНЗАКЦИИ
--------------------------------------
POST http://localhost:8080/api/v1/transactions
Тело запроса (JSON):
{
  "userUid": "<ВАШ userUid>",
  "walletUid": "<UID кошелька>",
  "amount": 100.00,
  "type": "TOPUP",
  "paymentRequestUid": "<UID платежного запроса>"
}
Ответ: массив созданных транзакций (скопируйте uid одной из них)

ШАГ 7. ПОИСК ТРАНЗАКЦИЙ ПО ФИЛЬТРАМ
--------------------------------------
GET http://localhost:8080/api/v1/transactions?user_uid=<ВАШ userUid>&type=TOPUP
Ответ: список транзакций

ШАГ 8. ПОЛУЧЕНИЕ СТАТУСА ТРАНЗАКЦИИ
--------------------------------------
GET http://localhost:8080/api/v1/transactions/<UID транзакции>/status
Ответ: объект с текущим состоянием транзакции
